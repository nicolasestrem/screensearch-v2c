## Release Hardening and Known Gaps

- Improve OCR robustness or acceptance criteria for underscore-delimited synthetic identifiers; the
  live app captured normalized evidence but not the exact random-suffix token.
- Refine no-evidence refusal rendering so cited-frame tiles are not shown when the answer states
  there is no direct evidence for the requested token.
- Complete the remaining P5 keyboard/focus matrix with a more reliable WebView2 focus probe or
  manual visual screenshots.
- Add or expose deterministic route error/loading-state triggers so the complete state matrix can be
  verified without fabricating product data.
- Add sidecar request logging that records the image path or image payload source for VLM requests
  without exposing unrelated screen contents.
- Consider using an isolated app-data override if the app gains supported configuration for it later;
  none was used here because the audit prompt explicitly forbade deleting or overwriting the real app
  data directory.
